# 研究想法总览 - 长时程规划智能体的 3 个候选方向

## 文件来源
- `~/Project/research/IDEA_REPORT.md`

## 总体方向
这份 idea report 的问题意识很清晰：
在 **training-free、API-only** 的前提下，长时程规划 agent 仍然会遇到：
- exploration complexity
- delayed credit assignment
- semantic drift
- 评估指标单一

## 3 个推荐 idea

### 1. Recovery as Plan Splicing
核心假设：
当计划中途失败时，做 **局部 subplan repair** 可能比 full replanning 更能保持全局目标一致性，也更省 token 和恢复时间。

为什么值得做：
- 很贴近实际控制问题
- 直观但尚未在 LLM agent 中被系统化
- 容易设计 pilot experiment

### 2. Counterfactual Failure Attribution
核心假设：
比起修复“最后一步错误”，如果 agent 能找出真正导致失败的 **earliest causally responsible step**，恢复效果会更好。

为什么值得做：
- 如果成立，是 recovery 的重要能力
- 如果不成立，也是对 LLM causal reasoning 的一个负结果
- intervention-based test 很适合写成有说服力的实验

### 3. Triggered Replanning via Hazard Scores
核心假设：
让 agent 在每一步前估计一个 hazard score，并在高风险时才触发 replanning，可能优于 never replan 或 always replan。

为什么值得做：
- 成本最低
- 最容易快速验证
- 即便结果为负，也具有研究价值

## 我的提炼
这 3 个 idea 可以看成同一条主线上的不同切面：
- **什么时候该重规划** → hazard score
- **失败后该修哪里** → counterfactual attribution
- **修的时候怎么修** → plan splicing
