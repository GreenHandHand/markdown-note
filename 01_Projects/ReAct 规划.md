# 规划

> [!definition|目标] 将规划内容整理为结构化信息，并进行自行探索的智能 Agent

## 主要结构

为了确保尽可能的确保 Agent 的自主性，不限制 Agent 的思路，这里采用**原子卡片 + 轻量连接**的方式组织结构化信息。目标是先不涉及复杂的图结构，先设计一个最小信息单元。

> [!definition|原子卡片] 原子卡片是用于记录一条信息的卡片，是一个基本节点，保存了一些基本信息。
> 一个原子卡片应当包含如下信息：
> - `title`：用于呈现给 Agent，简单描述了卡片的来源和用途
> - `content`：卡片的内容
> - `link`：卡片与其他卡片的连接信息，每个连接都是一个可以访问的节点
> - `type`：卡片的类型。
> 
> 除此之外，还有一些其他用于管理的信息与 metadata 等。

> [!note]
> 这里，`title` 和 `type` 一起构成了卡片的主键，我们不允许名称和类型均相同的卡片被创建。当 Agent 试图创建这样一个卡片时，我们会进行提示，并将系统中已有的卡片展示给 Agent，交由 Agent 进行决策。

> [!note]
> 对于 Agent 而言，它不需要知道自己目前在生成的是一张图、一棵树还是其他的数据结构，它只需要源源不断的生成新卡片、整理旧卡片即可。

> [!definition|type] 卡片类型
> **`type` 不是固定枚举，而是 Agent 在长期整理过程中逐步收敛出来的轻量概念集合。**
> 我们在这里定义一个卡片类型池，所有的原子卡片的类型都会引用自卡片类型池中的类型。在卡片类型池中，我们定义一个卡片类型如下：
> - `desc`：卡片类型的简单描述
> - `type_name`：卡片类型名称

> [!note]
> 实现上，这里提供一个 builtin type，可以称为 `untyped`，用于表述由于某些问题而失去 type 需要重新标记的卡片。使用内置的 reminder 在 untyped 的内容存在的时候，提示 Agent 进行处理。
