# Agent

## 定义

这里的定义来自 Google 的 Agents 白皮书：

> [!Definition] Agents
>  AI Agent 是一个**面向目标的完整应用**，它利用大型语言模型（LLM）作为推理引擎，在一个持续的**思考、行动、观察循环**中自主地完成任务。

Agnet 是自主的，可以在无需人工干预的情况下独立行动。Agent 在实现目标的方法是积极的主动的，即使没有来自人类的明确指令集，Agent 也可以推理出下一步应该做什么，以实现最终的目标。

Agents 的认知架构一般包含三个组件：
1. **模型**(Model)：作为 Agent 的中心推理引擎，负责处理信息、规划和决策。
	- Agent 所使用的模型可以是一个或者多个任意规模的语言模型。
	- 这些模型可以遵循基于指令的推理框架或者逻辑框架。
	- 这些模型可以是通用的、多模态的，也可以是针对特定架构的需求微调过的。
2. **工具**(Tools)：用于连接推理与现实世界，使 Agent 能够检索实时信息并采取行动。
	- 工具的形式多种多样，通常与 Web 中使用的 API 方法功能上一致 (GET, POST, PATCH, DELETE)。
3. **编排层**(The Orchestration Layer)：编排层描述了 Agent 如何接收信息、执行内部推理、并利用推理结果指导下一步行动或者决策。它负责护理规划、记忆、推理策略的执行，该层利用提示框架和推理技术 (CoT, ReAct) 来将复杂目标分解为步骤，并决定何时进行思考，何时使用工具。
	- 编排层的复杂度会按照 Agent 及所执行的任务而有很大的差异。

> [!note] Agents vs Models
> - **Models**：这里特指单一的 llm
> 	- 知识仅限于训练数据。
> 	- 对于用户的每一次查询，除非特别实现，否则模型不会管理上下文信息（聊天历史）
> 	- 没有原生的工具。
> 	- 没有逻辑层的实现，但是可以通过 CoT、ReAct 框架来引导模型思考。
> 	- 黑盒。
> - **Agents**：
> 	- 通过 Tools 与外部联系，以拓展知识。
> 	- 管理会话历史记录，用于支持用户查询和编排层中进行的多轮决策。*一轮决策*定义为交互双方之间的一次交互。
> 	- 使用工具。
> 	- 实现了逻辑层，并利用了 CoT、ReAct 等推理框架和 LangChain 等智能体框架。
> 	- 可以看到 Agent 的试错、思考、实现过程。

> [!tip] 推理框架简介
> 这里是现在流行的推理框架的简单介绍，下面的技术都是通过 Prompt 实现的。
> - CoT：思维链，通过中间步骤来引导 LLM 进行思考。
> - ReAct：为 LLM 提供了一种思维过程策略，使得可以对用户查询进行推理并采取行动。
> - ToT：思维树，推广了思维链的提示，运行模型探索各种中间步骤的思维链，适合探索或者前瞻性任务。

> [!example] Agent 的一个流程
> ![[98_Assets/Agent 调研-1.png]]
> 1. 用户向 Agent 发送 Query
> 2. Agent 开始一个 ReAct 序列
> 3. Agent 向模型提供 prompt，要求其生成下一个 ReAct 步骤和对应的输出：
> 	- **问题**：来自用户查询的输入问题，随 Prompt 一同提供
> 	- **Loop**：
> 		1. **思考**：模型下一步应该做什么的思考
> 		2. **行动**：模型关于下一步采取什么行动的决定。
> 			- 这里可能发生工具选择，例如行动可以是从 `[Flights, Search, Code, None]` 中选择一个。
> 		3. **行动输入**：模型准备工具的输入。
> 		4. **观察结果**：获取行动的结果。
> - **最终答案**：ReAct 循环结束，模型返回最终答案。

## 分类

我们可以将 Agent 划分为几个宽泛的层级，其中每一个层级都建立在上一级能力的基础上。
1. The Core Reasoning System：单一的 LLM。
	- 在这一层中，Agent 通过了广泛的训练，拥有了解决问题、解释概念的能力，但是完全缺乏实时感知能力。对于其训练数据之外的任何事件或事实，它是不知道的。
	- 例如，它可以解释职业棒球的规则和纽约洋基队的完整历史。但如果你问：“昨晚洋基队比赛的最终比分是多少？”，它将无法回答。
2. The Connected Problem-Solver：引入 Tools 之后的 Agent。
	- 在这一层中，LLM 通过连接并使用外部的工具，成为了一个功能性的 Agent。
	- 该智能体现在可以会在之前的问题，当它识别到任务中的实时需求，它会在行动步骤中调用工具，例如 Google API，并观察搜索结果，将这一事实综合成一个答案。
3. The Strategic Problem-Solver：二级标志着能力的显著扩展，从执行简单任务转向战略性规划复杂、多环节的任务。
	- 在这一层中，Agent 的准确性依赖于一个精心管理、聚焦、高质量的上下文，防止 LLM 过载并确保其高效性能。
	- 可以参考现在主流的各种编程 Agent，针对一个需求，会先规划任务，然后一一执行。
4. The Collaborative Multi-Agent System：多智能体系统。
	- 在这一层次，我们不再构建单一、全能的超级智能体，而是转向一个协同工作的专家团队模型。
	- 在这里，智能体会将其他智能体视为工具。当一个项目经理智能体接收到一项任务时，它不会独自工作，而是通过为团队中的专业智能体创建新任务来行动。这很像现实生活中的运作方式。
5. The Self-Evolving System：
	- 在这一级别中，一个 Agent 系统会自动识别自身能力的不足，并动态创建新的工具甚至新的智能体来填补空白。
	- 从使用固定的资源转向使用拓展资源。
